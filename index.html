Compose beta

A simple WYSIWYG editor focused on content.

Demo & Features

This is the actual content editor you're looking at. Go on, start editing this text.

A good content editor should provide simple tools for formating but most importantly it should emphasis the content. After all, that's what you are doing with it, writting.

Formating

Formating tools are kept out of the way until you need them. Try selecting some text to see how it works.

Compose has built-in Markdown support so you can format your text with natural synthax. Start titles with # or lists with *, etc.

Notes on embedding Compose

Please note that Compose is beta at this point. Documentation is still inexistent and some core tools are missing. This page will be updated as the project matures.

Compose is not really meant to be dumped on top of a project without customization. The whole point of Compose is to show the writer how his content will be displayed in the end; this means you should at least style the editor to look like the final content. Usually this only involves loading a css file.
Apart from this, Compose is composed (haha) of a js file and a css file.

Browser support & dependency

At this point, wide browser support is not a priority for Compose. It works in the latest (and not-so-latest) versions of Chrome & Firefox. It should work in Safari & Opera but this hasn't been tested yet. IE versions above 9 will be supported any time soon, versions 9 or below are not on the schedule right now.

Compose uses jQuery for its internal DOM manipulation and tools. However it only uses a rather small and stable portion of jQuery, which means any recent version of jQuery will do. While I don't feel it's problematic right now, the jQuery dependecy might be dropped at some point in the future.

Tools & customization

Again, Compose is meant to be simple for the end user — this includes not clutering the interface with tools he'll never use. Compose ships with a set of tools that include two title levels, bold, italics, and quotes. Future tools will include links and maybe images.

You understandbly might feel the need to add other tools. While the tools API isn't very mature, it's actually quite easy to do. As of now, you should read the corresponding source code. A better documentation will come at some point in the future.

Styling the tools is done via css, just override the default styles that are shipped with Compose.

Markdown support can be turned off quite easily. Look at the code until it's documented.
